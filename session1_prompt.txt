════════════════════════════════════════════════════════════════════════════════
CSV FORMAT  (ground truth from real files)
════════════════════════════════════════════════════════════════════════════════

- Line endings: CRLF (\r\n) — must be preserved on save
- No header row. Row 0 is not special.
- Every row has exactly 17 comma-separated fields (columns 0–16)
- COMMENT rows: column 0 ≠ "#NOTE"
    → Preserve the entire raw line verbatim
    → Blank separator rows: ,,,,,,,,,,,,,,,, (16 commas, 17 empty fields)
- DATA rows: column 0 = "#NOTE", always exactly 17 fields
- File named set_nnnn.csv where nnnn is a 4-digit preset number
  with leading zeros e.g. set_0001.csv


════════════════════════════════════════════════════════════════════════════════
COLUMN SCHEMA  (17 columns, indices 0–16)
════════════════════════════════════════════════════════════════════════════════

col 0:  sentinel        Fixed string "#NOTE". Never shown as a column in UI.

col 1:  midi_note       int, 0–127.
                        Display as "60  C4" (number + two spaces + note name).
                        Note name helper:
                          const NAMES = ['C','C#','D','D#','E','F','F#','G',
                                         'G#','A','A#','B'];
                          const noteName = n =>
                            NAMES[n % 12] + (Math.floor(n / 12) - 1);

col 2:  midi_channel    int, 0–16.
                        0–15 = specific MIDI channel.
                        16   = Omni (respond to any channel).
                        Display 16 as "Omni" in the UI.

col 3:  action_type     Enum. Stored and serialised as full label string:
                          "01 - Play Note"
                          "02 - Trigger Type 1"
                          "03 - Trigger Type 2"
                          "04 - Trigger Type 3"
                          "05 - Stop Track"
                          "06 - Stop All"
                          "07 - Load Preset"
                        Parser reads only the leading 2-digit number.
                        Always serialise the full label string exactly.

col 4:  track_preset    int.
                        Actions 01–05: track number, 1–4095.
                        Action 07:     preset number, 1–9999.
                        Action 06:     empty string "".

col 5:  pitch_offset    int, -700 to +700 (cents; 100 cents = 1 semitone).
                        Empty string "" for actions 05, 06, 07.

col 6:  attack_ms       int, 0–99999 (milliseconds).
                        Empty string "" for actions 05, 06, 07.

col 7:  release_ms      int, 0–99999 (milliseconds).
                        Empty string "" for action 06.

col 8:  loop_flag       int, 0 or 1. Render as checkbox. Label: "Loop".

col 9:  lock_flag       int, 0 or 1. Render as checkbox. Label: "Lock".

col 10: pitch_bend_flag int, 0 or 1. Render as checkbox. Label: "Pitch Bend".

col 11: min_velocity    int, 0–127.

col 12: max_velocity    int, 0–127.
                        Validate: min_velocity ≤ max_velocity.

col 13: min_vel_gain    float, -100.0 to 0.0 (dB).
                        Validate: min_vel_gain ≤ max_vel_gain.
                        Equal values valid (= fixed volume).

col 14: max_vel_gain    float, -100.0 to 0.0 (dB).

col 15: balance         int, 0–127.
                        0=full left, 64=center, 127=full right.
                        Render as horizontal range slider with L/R labels
                        plus numeric input for precision entry.
                        Visual centre detent indicator at value 64.

col 16: comment         Free text. Preserve if present. Always empty in
                        practice on #NOTE rows but do not discard.

ACTIVE COLUMNS BY ACTION TYPE
════════════════════════════════════════════════════════════════════════════════

All 17 columns always written on save. Empty string "" for inapplicable cols.
In UI: disable and visually grey out inapplicable cells.

  Actions 01–04:    all columns editable
  Action 05:        editable: cols 1,2,3,4,7 — disabled: cols 5,6,8–16
  Action 06:        editable: cols 1,2,3     — disabled: cols 4–16
  Action 07:        editable: cols 1,2,3,4   — disabled: cols 5–16

CROSS-FIELD VALIDATION
════════════════════════════════════════════════════════════════════════════════

- min_velocity ≤ max_velocity  (both 0–127)
- min_vel_gain ≤ max_vel_gain  (both -100.0 to 0.0; equal values valid)
- All per-column bounds as specified above
- Save blocked when any data row has validation errors


════════════════════════════════════════════════════════════════════════════════
DOCUMENT STRUCTURE
════════════════════════════════════════════════════════════════════════════════

The document has two independent sections:

────────────────────────────────────────
1. COMMENTS SECTION
────────────────────────────────────────

All non-#NOTE rows from the imported file, in original order.
Rendered as a plain <textarea> in a collapsible panel above the tree.
Panel header shows line count and collapse/expand chevron.

The "Do not edit this block" legend found in Google Sheets exports
appears here as ordinary text. User may delete it. Do NOT auto-generate
this block on save.

Serialisation:
  Non-empty line → text in col 0 + 16 trailing commas
  Empty line     → 16 commas (blank separator row)

────────────────────────────────────────
2. NOTE ASSIGNMENTS — THREE-LEVEL TREE
────────────────────────────────────────

Structure: Channel → Note → Actions

LEVEL 1 — CHANNEL ROWS (17 rows total, always visible)
  - Omni (channel 16) shown first, at the top
  - Channels 0–15 shown below in numeric order
  - Each channel row shows:
      • Expand/collapse chevron (left)
      • Channel label: "Ch 0", "Ch 1" ... "Ch 15", "Omni"
      • Count badge: "N notes, M actions"
      • Colour dot summary: one dot per unique action type present
        in this channel's assignments
  - Default state on load: ALL channel rows collapsed
  - Channels with no assignments are still shown but badge reads "empty"

LEVEL 2 — NOTE ROWS (visible when parent channel is expanded)
  - Show ONLY notes that have at least one action assigned to this channel
  - Each note row shows:
      • Expand/collapse chevron (left, indented)
      • Note label: "60  C4" (number + name)
      • Count badge: "N actions"
      • Colour dots: one per action row, coloured by action type
      • [+ Add Action] button — disabled when note has 8 actions
      • [× Delete Note] button — removes all actions for this
        note/channel combination, with confirmation dialog
  - Default state on load: all note rows collapsed

LEVEL 3 — ACTION ROWS (visible when parent note is expanded)
  - Compact grid row showing all field values
  - Drag handle on left for drag-and-drop reordering within the note group
    (use @dnd-kit/sortable for drag and drop)
  - Clicking a row selects it and populates the Detail Panel
  - Left border stripe coloured by action type
  - Disabled cells visually greyed out per action type rules

GRID COLUMNS (displayed left to right, col 0 sentinel never shown):
  [drag] | Chan | Note | Action | Track | Pitch | Atk | Rel |
  Loop | Lock | Bend | MinV | MaxV | MinG | MaxG | Bal | Cmt

Sticky/frozen on horizontal scroll: Chan, Note, Action always visible.

COLUMN HEADERS (abbreviated):
  Chan | Note | Action | Track | Pitch | Atk | Rel | Loop | Lock |
  Bend | MinV | MaxV | MinG | MaxG | Bal | Cmt

────────────────────────────────────────
3. DETAIL PANEL (below the tree, always visible)
────────────────────────────────────────

Shows full labelled form for the currently selected action row.
Updates live — changes reflect immediately in the grid row.
Editing a grid cell also updates the detail panel.
When no row selected: show "Select an action row to edit its details."

Form layout (grouped by meaning):

  Row 1:  Channel [select, shows "Omni" for 16]
          Action  [select from enum]

  Row 2:  Track / Preset [number — label changes by action type]

  Row 3:  Attack  [number, ms] — hidden for actions 05,06,07
          Release [number, ms] — hidden for action 06

  Row 4:  [□ Loop]  [□ Lock]  [□ Pitch Bend]

  Row 5:  Velocity  Min [____] ———range——— Max [____]
          — hidden for actions 05,06,07

  Row 6:  Gain (dB) Min [____] ░░░░░░░░░░ Max [____]

  Row 7:  Balance  L ————●———— R  [____]

  Row 8:  Comment [text input]

────────────────────────────────────────

  Channel: [select 0–16, shows "Omni" for 16]

5. NEW PRESET (toolbar button)
────────────────────────────────────────

  - Prompts for a 4-digit preset number (1–9999)

  - Creates a completely empty document:
      • Empty comments section

  - Sets the preset number for save filename

────────────────────────────────────────
6. FILL SEQUENTIAL NOTES (toolbar button → modal dialog)
────────────────────────────────────────

  From note:          [number input 0–127]  [live note name]
  To note:            [number input 0–127]  [live note name]
  Channel:            [select 0–16]
  Action type:        [select from enum]
  Start track:        [number input 1–4095]

════════════════════════════════════════════════════════════════════════════════
REACT STATE ARCHITECTURE
════════════════════════════════════════════════════════════════════════════════

Use useReducer at the App level for all document state.
Use useState for purely local UI state (expand/collapse, selected row, modals).

Document state shape:
  {
    presetNumber: number,          // 1–9999
    comments: string[],            // one string per comment line
    // Keyed by "channel:noteNum" e.g. "16:60"
    noteActions: Map<string, DataRow[]>
  }

UI state (local useState, NOT in the reducer):
  expandedChannels: Set<number>    // which channel rows are expanded
  expandedNotes: Set<string>       // which "channel:noteNum" notes are expanded
  selectedRowKey: string | null    // currently selected action row
  undoStack: []                    // array of document state snapshots
  redoStack: []                    // array of document state snapshots

CRITICAL: expandedChannels and expandedNotes must be plain React useState
with Set replacement on every update (never mutate the Set in place):
  const [expandedChannels, setExpandedChannels] = useState(new Set());
  
  function toggleChannel(ch) {
    setExpandedChannels(prev => {
      const next = new Set(prev);
      next.has(ch) ? next.delete(ch) : next.add(ch);
      return next;
    });
  }

This pattern is mandatory. Do not use useRef, do not mutate Sets or Maps
directly, always return new Set/Map instances from state updaters.

CELL EDITOR TYPES
════════════════════════════════════════════════════════════════════════════════
